# 리버스 쉘(Reverse Shell) 제작 계획

## 프로그램 목표

일반적인 Bind Shell은 피해자의 PC에서 방화벽으로 공격자의 PC 접속이 막힐 수 있다.
따라서, Reverse Shell로 공격자의 PC에서 서버를 열고 피해자가 직접 접속하도록 한다. 


## 요구사항

### Client 프로그램(피해자 PC)

공격자의 명령어를 받아 수행 후 공격자에게 결과를 전송한다.
- 공격자의IP와 지정된 포트로 연결을 시도한다.
- 연결이 성공하면 공격자의 명령어를 수신한다.
- 수신된 명령어를 시스템 쉘에서 실행한다.
- 실행 결과를 공격자에게 전송한다.

### Server 프로그램(공격자 PC)

피해자의 접속을 기다리고 연결이 된 후 명령어를 보낸 후 결과를 출력한다.
- 자신의 IP와 지정된 포트로 소켓을 생성해야한다.
- 피해자의 접속을 기다려야한다.
- 피해자의 접속을 수락한다.
- 피해자에게 shell, cmd 명령어를 보낼 수 있어야한다. 
- 피해자에게 서 온 응답을 화면에 출력해야한다.
